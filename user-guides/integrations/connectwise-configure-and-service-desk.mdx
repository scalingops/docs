# Configure ConnectWise Service Desk Integration

This guide explains how to configure ConnectWise Manage service desk settings for ticket synchronization and board management in MSPortal.

## Prerequisites

- ConnectWise Manage integration must be enabled
- You must have "Manage Integrations" permission
- ConnectWise API credentials must be configured
- At least one service board must exist in ConnectWise

## Overview

The ConnectWise service desk integration allows you to:
- Sync tickets between MSPortal and ConnectWise
- Configure which boards are visible to users
- Set default boards for new tickets
- Map ticket statuses and priorities
- Control ticket synchronization behavior

## Configuration Steps

### 1. Navigate to Integration Settings

Go to [https://msportal.ai/settings/integrations](https://msportal.ai/settings/integrations)

### 2. Access Ticket Settings

Click on **Ticket Settings** in the integration options.

![](https://ajeuwbhvhr.cloudimg.io/https://colony-recorder.s3.amazonaws.com/files/2025-09-04/f4249250-8b3e-4451-b316-acc87d537087/ascreenshot.jpeg?tl_px=743,116&br_px=2463,1077&force_format=jpeg&q=100&width=1120.0&wat=1&wat_opacity=0.7&wat_gravity=northwest&wat_url=https://colony-recorder.s3.us-west-1.amazonaws.com/images/watermarks/FB923C_standard.png&wat_pad=557,277)

### 3. Enable Ticket Synchronization

Toggle the **Enable Ticket Sync** switch to activate synchronization between MSPortal and ConnectWise.

![](https://ajeuwbhvhr.cloudimg.io/https://colony-recorder.s3.amazonaws.com/files/2025-09-04/aa28deea-900a-497a-9fca-bb8ab2ca1233/ascreenshot.jpeg?tl_px=0,0&br_px=1719,961&force_format=jpeg&q=100&width=1120.0&wat=1&wat_opacity=0.7&wat_gravity=northwest&wat_url=https://colony-recorder.s3.us-west-1.amazonaws.com/images/watermarks/FB923C_standard.png&wat_pad=522,111)

### 4. Select Display Boards

Choose which ConnectWise boards should be visible to MSPortal users.

![](https://ajeuwbhvhr.cloudimg.io/https://colony-recorder.s3.amazonaws.com/files/2025-09-04/d2adb09a-1037-4cd8-b80b-281bb1e87f49/ascreenshot.jpeg?tl_px=260,0&br_px=1979,961&force_format=jpeg&q=100&width=1120.0&wat=1&wat_opacity=0.7&wat_gravity=northwest&wat_url=https://colony-recorder.s3.us-west-1.amazonaws.com/images/watermarks/FB923C_standard.png&wat_pad=524,202)

Select the boards you want to display:
- **Service Board** - For standard service tickets
- **Professional Services** - For project-related tickets
- **Integration** - For integration and API issues
- **Support** - For help desk requests

You can select multiple boards by clicking each one. The selected count will display (e.g., "2 selected").

### 5. Set Default Service Board

Select the default board where new tickets created in MSPortal will be sent.

![](https://ajeuwbhvhr.cloudimg.io/https://colony-recorder.s3.amazonaws.com/files/2025-09-04/5735bc6a-cd3b-4da7-a484-f05e7590c3de/ascreenshot.jpeg?tl_px=177,63&br_px=1896,1024&force_format=jpeg&q=100&width=1120.0&wat=1&wat_opacity=0.7&wat_gravity=northwest&wat_url=https://colony-recorder.s3.us-west-1.amazonaws.com/images/watermarks/FB923C_standard.png&wat_pad=524,277)

This setting determines:
- Where tickets created through MSPortal are placed
- The default board for ticket templates
- The initial board for automated ticket creation

#### Waiting Status
Select the status to apply when tickets are put in a waiting state:

![](https://ajeuwbhvhr.cloudimg.io/https://colony-recorder.s3.amazonaws.com/files/2025-09-04/88f39ec5-e86c-4285-9915-5bd2be1c5930/ascreenshot.jpeg?tl_px=245,70&br_px=1965,1031&force_format=jpeg&q=100&width=1120.0&wat=1&wat_opacity=0.7&wat_gravity=northwest&wat_url=https://colony-recorder.s3.us-west-1.amazonaws.com/images/watermarks/FB923C_standard.png&wat_pad=524,276)

#### Client Response Status

- Select a specific status like "Client Responded"

#### Excluded Statuses
Select statuses that should NOT sync between systems:

![](https://ajeuwbhvhr.cloudimg.io/https://colony-recorder.s3.amazonaws.com/files/2025-09-04/9e0e323f-1b0f-4699-b8bc-cd08074d25d6/ascreenshot.jpeg?tl_px=243,230&br_px=1962,1191&force_format=jpeg&q=100&width=1120.0&wat=1&wat_opacity=0.7&wat_gravity=northwest&wat_url=https://colony-recorder.s3.us-west-1.amazonaws.com/images/watermarks/FB923C_standard.png&wat_pad=524,277)

- Closed tickets
- Cancelled tickets
- Internal-only statuses

### 7. Configure Priority Settings

Set up priority user handling for VIP or high-priority clients:

![](https://ajeuwbhvhr.cloudimg.io/https://colony-recorder.s3.amazonaws.com/files/2025-09-04/0b9ee90a-6dba-42a3-a5c0-8d37d02b25d1/ascreenshot.jpeg?tl_px=178,223&br_px=1897,1184&force_format=jpeg&q=100&width=1120.0&wat=1&wat_opacity=0.7&wat_gravity=northwest&wat_url=https://colony-recorder.s3.us-west-1.amazonaws.com/images/watermarks/FB923C_standard.png&wat_pad=524,277)

Options include:
- **Critical** - For emergency issues
- **High** - For VIP clients
- **Medium** - Standard priority
- **Low** - Non-urgent requests

### 8. Save Configuration

Click **Save Settings** to apply all configuration changes.

![](https://ajeuwbhvhr.cloudimg.io/https://colony-recorder.s3.amazonaws.com/files/2025-09-04/a8bb29ef-503d-4e10-b6d1-5c84c2bfe664/ascreenshot.jpeg?tl_px=676,243&br_px=2396,1205&force_format=jpeg&q=100&width=1120.0&wat=1&wat_opacity=0.7&wat_gravity=northwest&wat_url=https://colony-recorder.s3.us-west-1.amazonaws.com/images/watermarks/FB923C_standard.png&wat_pad=524,507)

## Advanced Settings

### Sync Frequency
- Tickets sync every 5 minutes by default
- Real-time sync available for critical updates
- Manual sync can be triggered from the Tickets page

### Field Mapping
MSPortal automatically maps common fields:
- Title/Summary
- Description
- Priority
- Status
- Assigned technician
- Company
- Contact

### Custom Fields
Custom fields created in ConnectWise can be:
- Mapped to MSPortal ticket fields
- Displayed in ticket templates
- Used in automation rules

## Testing the Integration

After configuration:

1. Create a test ticket in MSPortal
2. Verify it appears in the correct ConnectWise board
3. Update the ticket in ConnectWise
4. Confirm changes sync back to MSPortal
5. Test status transitions and priority changes

## Troubleshooting

### Tickets Not Syncing
- Verify ticket sync is enabled
- Check ConnectWise API credentials
- Ensure boards are properly selected
- Review excluded status settings

### Wrong Board Assignment
- Confirm default service board is set correctly
- Check ticket template board settings
- Verify user permissions for selected boards

### Status Mapping Issues
- Review status mappings match ConnectWise statuses
- Ensure statuses aren't in excluded list
- Check for custom status configurations

## Best Practices

- **Limit Display Boards:** Only show boards relevant to MSPortal users
- **Use Status Exclusions:** Prevent closed tickets from syncing unnecessarily
- **Configure Priority Users:** Set up VIP handling for important clients
- **Regular Testing:** Periodically test sync to ensure proper operation
- **Document Mappings:** Keep a record of status and field mappings
