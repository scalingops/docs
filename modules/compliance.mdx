## Overview

The Compliance module enables comprehensive management of compliance requirements across multiple frameworks. Schedule compliance runs, track check results, manage issues, and maintain audit trails for all your clients.

## Compliance Frameworks

### Supported Frameworks
- **HIPAA**: Healthcare compliance
- **PCI DSS**: Payment card industry standards
- **SOC 2**: Service organization controls
- **GDPR**: Data privacy regulations
- **NIST**: Cybersecurity framework
- **ISO 27001**: Information security
- **Custom Frameworks**: Define your own

### Framework Configuration
Each framework includes:
- Control requirements
- Check definitions
- Evidence requirements
- Remediation guidance
- Reporting templates

## Compliance Runs

### Running Compliance Checks
<Steps>
  <Step title="Select Framework">
    Choose compliance framework to assess
  </Step>
  <Step title="Choose Scope">
    Select company and systems to check
  </Step>
  <Step title="Configure Checks">
    Select specific checks or run all
  </Step>
  <Step title="Execute Run">
    Run checks manually or schedule
  </Step>
  <Step title="Review Results">
    Analyze findings and issues
  </Step>
</Steps>

### Run Status
- **Scheduled**: Planned future run
- **Running**: Currently executing
- **Completed**: Successfully finished
- **Failed**: Errors during execution
- **Cancelled**: Manually stopped

### Scheduling
Set up automated compliance runs:
- Daily, weekly, monthly, quarterly
- Custom schedules
- Blackout periods
- Notification settings

## Issue Management

### Issue Tracking
When checks fail, issues are created:
- Issue description
- Severity level (Critical, High, Medium, Low)
- Affected systems
- Remediation steps
- Due date for resolution

### Issue Workflow
<Steps>
  <Step title="Detection">
    Issue identified during compliance run
  </Step>
  <Step title="Assignment">
    Assign to responsible party
  </Step>
  <Step title="Remediation">
    Implement fixes
  </Step>
  <Step title="Verification">
    Verify issue resolved
  </Step>
  <Step title="Closure">
    Document and close issue
  </Step>
</Steps>
